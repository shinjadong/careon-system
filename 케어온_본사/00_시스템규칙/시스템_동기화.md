# [[시스템_동기화]]

## 1. 개념

**GitHub, Obsidian, Notion은 동일한 지식을 다른 도구로 관리하는 동질 개념입니다.**

- **GitHub**: 버전 관리, 백업, 협업, 외부 공개
- **Obsidian**: 로컬 작업, 링크 기반 탐색, 빠른 편집
- **Notion**: 데이터베이스, 팀 협업, 시각화, 외부 공유

## 4. 파일 네이밍 규칙

모든 플랫폼에서 **동일한 네이밍 규칙**을 따릅니다.

### 형식
```
[[폴더명_핵심키워드]]
```

### 예시
- Obsidian: `[[마케팅_블로그템플릿]].md`
- GitHub: `마케팅_블로그템플릿.md`
- Notion: "마케팅_블로그템플릿" 페이지

---

## 5. 메타데이터 규칙

모든 문서는 **동일한 메타데이터**를 포함합니다.

### Obsidian & GitHub (YAML Frontmatter)
```yaml
---
분류: [폴더 카테고리]
설명: [파일 내용 1줄 요약]
상위문서: [[상위개념_링크]]
작성일: 2025-MM-DD
기존출처: Notion (URL있으면 기입)
---
```

### Notion (Properties)
- **분류**: Select
- **설명**: Text
- **상위문서**: Relation
- **작성일**: Date
- **원본출처**: URL

## 6. 동기화 작업 프로세스

### A. Notion → Obsidian (마이그레이션)

**시기**: 초기 데이터 이관 시

**절차**:
1. Notion MCP로 페이지 읽기
2. 내용 분석 후 적절한 폴더 분류
3. 파일명을 `[[폴더명_핵심키워드]]` 형식으로 변경
4. YAML Frontmatter 추가 (기존출처: Notion URL 포함)
5. Obsidian Vault에 저장

**예시**:
```markdown
---
분류: 마케팅
설명: 블로그 작성 템플릿 및 가이드
상위문서: [[마케팅_블로그오토메이션]]
작성일: 2025-10-22
기존출처: https://www.notion.so/294b8018d368800a932cc47712bfe94a
---

# [[마케팅_블로그작성요령]]

...내용...
```

### B. Obsidian → GitHub (백업/공개)

**시기**: 매일 또는 주요 변경 시

**절차**:
```bash
cd /home/tlswkehd/careon
git add .
git commit -m "docs: 문서 업데이트 - [변경 내용 요약]"
git push origin main
```

**자동화 (선택)**:
```bash
# cron으로 매일 자정 자동 커밋
0 0 * * * cd /home/tlswkehd/careon && git add . && git commit -m "docs: 자동 백업" && git push
```

### C. Obsidian → Notion (협업 문서)

**시기**: 팀 협업이 필요한 문서만 선택적으로

**절차**:
1. 중요 문서 선별
2. Notion MCP로 페이지 생성
3. 메타데이터 매핑
4. 양방향 링크 유지 (Obsidian에 Notion URL 기록)

**주의**:
- 모든 문서를 Notion에 복사하지 않음
- 협업이 필요한 문서만 선택적 동기화
- Notion은 보조 도구로 활용

## 7. 충돌 해결 원칙

### 원칙 1: Obsidian이 항상 우선
Obsidian과 다른 플랫폼 간 내용이 다를 경우, **Obsidian 내용이 정답**입니다.

### 원칙 2: 날짜 기반 버전 관리
```
마케팅_블로그템플릿.md (최신)
마케팅_블로그템플릿_20251118.md (백업)
```

### 원칙 3: 충돌 시 수동 병합
자동 병합하지 않고 사용자가 직접 확인 후 병합합니다.

## 10. 도구별 특징 및 활용법

### Obsidian
- **장점**: 빠른 로컬 검색, 그래프 뷰, 링크 기반 탐색
- **활용**: 일상 작업, 아이디어 정리, 문서 간 연결
- **플러그인**: Dataview, Calendar, Mind Map

### GitHub
- **장점**: 버전 관리, 백업, 외부 공개, 협업
- **활용**: 백업 저장소, 문서 버전 추적
- **기능**: Issues, Projects, Wiki

### Notion
- **장점**: 데이터베이스, 팀 협업, 시각적 정리
- **활용**: 팀 공유 문서, 프로젝트 관리, 외부 공유
- **기능**: 칸반 보드, 타임라인, 갤러리
