# [[시스템_네이밍규칙]]

## 1. 기본 원칙 (Philosophy)
1. **직관성:** 엑셀 헤더처럼 누구나 제목만 보고 내용을 유추할 수 있어야 한다.
2. **구조화:** 모든 문서는 정해진 폴더 구조 중 하나에 속해야 한다.
3. **의존성:** 상위 목적(비전/미션) 없이 존재하는 문서는 없다.

## 2. 파일 네이밍 컨벤션 (Naming Convention)

**형식:** `[[폴더명_핵심키워드]]`
- 띄어쓰기 없이 연결하거나, 가독성을 위해 언더바(`_`)를 사용한다.
- 대괄호 `[[ ]]`는 옵시디언 링크 기능을 위해 필수적으로 사용한다.

**[예시]**
- (O) `[[마케팅_블로그템플릿]]` : 마케팅 폴더의 블로그 템플릿
- (O) `[[고객_CCTV_3-2_캡스위약금]]` : 고객 폴더의 3-2번 타겟
- (X) `20251118_회의록` : 분류가 없음 (-> `[[기록_251118_신예준통화]]` 로 변경)
